## 依赖升级到最新版本

目标：把 [slack-frontend/package.json](slack-frontend/package.json) 中依赖升级到最新，保证 `pnpm build`、`pnpm lint` 全部通过。所有命令均在 `slack-frontend/` 下执行（用 pnpm）。

### 前置事实

- 运行时已切到 Node LTS `24.16.0`，pnpm `11.7.0` —— 满足 ESLint 10 / `@eslint/compat 2` 引擎要求。
- `typescript-eslint@8.61` 要求 `typescript >=4.8.4 <6.1.0`，故 TS `6.0.3` 在允许范围内。
- ESLint 10 已被 `eslint-config-next@16.2.9`、`typescript-eslint@8.61`、`eslint-plugin-react-hooks@7.1.1` 支持。
- `.husky` 无需改动：`husky` 不是依赖（hooks 由 [scripts/setup-git-hooks.mjs](slack-frontend/scripts/setup-git-hooks.mjs) 设 `core.hooksPath` 接管），hook 脚本按命令名调用 `pnpm lint-staged` / `npx commitlint`，会自动用上新版本；`.lintstagedrc.cjs` / `.commitlintrc.cjs` 配置格式在 lint-staged 17 / commitlint 21 下仍兼容。lint-staged 17 需 `Node >=22.22.1`、commitlint 21 需 `Node >=22.12.0`，Node 24.16.0 满足。

### 步骤 1：改写 package.json

对 [slack-frontend/package.json](slack-frontend/package.json) 做以下调整（沿用各条目现有的精确/caret 前缀风格）：

dependencies:

- `@heroui/react` `@heroui/styles` 3.0.3 → 3.2.1
- `@teispace/next-themes` 0.4.1 → 2.0.2
- `lucide-react` → 1.21.0
- `next` 16.2.4 → 16.2.9
- `next-intl` → 4.13.0
- `react` `react-dom` 19.2.5 → 19.2.7
- `react-resizable-panels` → 4.11.2
- `clsx` 保持 2.1.1（已最新）
- 删除 `intl-messageformat`（代码未引用，next-intl 也不再依赖）

devDependencies:

- `eslint` 9.25.1 → 10.5.0、`@eslint/js` → 10.0.1、`@eslint/compat` → 2.1.0、`@eslint/eslintrc` → 3.3.5
- `typescript` 5.9.3 → 6.0.3
- `@types/node` 20.x → `^24`（对齐运行时 Node 24.16.0）
- `@types/react` → 19.2.17（`@types/react-dom` 19.2.3 已最新）
- `@next/eslint-plugin-next` `eslint-config-next` → 16.2.9
- `@tailwindcss/postcss` `tailwindcss` → 4.3.1
- `@typescript-eslint/eslint-plugin` `@typescript-eslint/parser` → 8.61.1
- `eslint-config-prettier` → 10.1.8、`eslint-plugin-prettier` → 5.5.6、`eslint-plugin-react-hooks` → 7.1.1、`eslint-plugin-unused-imports` → 4.4.1
- `@commitlint/cli` `@commitlint/config-conventional` 20 → 21
- `commitizen` → 4.3.2、`cz-customizable` → 7.5.4
- `globals` 16 → 17、`lint-staged` 16 → 17、`postcss` → 8.5.15、`prettier` → 3.8.4
- 删除 `eslint-plugin-node`（`eslint.config.mjs` 未引用，已停止维护）
- 保持不变（已最新）：`eslint-plugin-import`、`eslint-plugin-jsx-a11y`、`eslint-plugin-react`、`tailwind-variants`

### 步骤 2：重新安装

- 运行 `pnpm install`，确认无 peer 依赖冲突报错。

### 步骤 3：构建验证（最大风险点 TS 6.0）

- 运行 `pnpm build`，修复 TypeScript 6.0 / Next 16.2.9 可能引入的类型或语法报错。

### 步骤 4：Lint 验证（ESLint 10 成套升级）

- 运行 `pnpm lint`，验证 [slack-frontend/eslint.config.mjs](slack-frontend/eslint.config.mjs) 中 `FlatCompat` / `fixupConfigRules` / `fixupPluginRules` 在 ESLint 10 + `@eslint/compat 2` 下仍可用；如有 API 变化按需调整配置。

### 步骤 5：主题库回归（next-themes 2.0）

- 检查 [slack-frontend/app/providers.tsx](slack-frontend/app/providers.tsx) 与 [slack-frontend/components/theme-switch.tsx](slack-frontend/components/theme-switch.tsx) 中 `ThemeProvider` / `useTheme` / `ThemeProviderProps` 在 `@teispace/next-themes@2.0.2` 下类型与运行均正常（已随 build 一并校验）。

### 步骤 6：开发态冒烟

- `pnpm dev` 启动后确认首页、工作台、主题切换、中英文切换无运行时报错。

### 步骤 7：Git hooks 验证（lint-staged 17 / commitlint 21 大版本）

- 验证 [slack-frontend/.husky/pre-commit](slack-frontend/.husky/pre-commit)：暂存一个 `.tsx` 文件后触发，确认 `pnpm lint-staged` 在 v17 下能正常跑 `eslint --fix` + `prettier --write`。
- 验证 [slack-frontend/.husky/commit-msg](slack-frontend/.husky/commit-msg)：用一条不规范 commit message 触发，确认 commitlint 21 能正确拦截。
- hook 脚本与 `.lintstagedrc.cjs` / `.commitlintrc.cjs` 预计无需改动；仅在 v17/v21 报配置不兼容时才调整。

### 风险与回退

- 高风险项：TypeScript 6.0、ESLint 10 三件套、`@teispace/next-themes` 2.0、commitlint/lint-staged/globals 大版本。
- 若某个 major 升级导致无法在合理改动内修复，将单独回退该依赖到当前可用的最高兼容版本，并在总结中说明。
