## 二、「100% 参考」的实现边界（技术与合规）

- **目标**：布局结构、层级、间距、字号层级、侧边栏宽度、顶栏/底栏、弹窗样式、空状态与插图占位，与参考工作台在**同一设计语言**下一一对应；交互以「可点开关、不切真实数据」为主。
- **验证方式**：按你提供的截图资源做逐项对照清单（侧边栏条目、图标位、编辑器三栏比例、右下角悬浮按钮堆叠、右侧 AI 面板文案与快捷链等）；开发中可用浏览器 DevTools 量取对标站（自用学习场景）的尺寸与颜色，写入设计 Token。
- **说明**：完全一致依赖「可测量的设计规范」（色值、圆角、阴影、栅格）；若仅有截图，可先定 **Design Token 文件 + Storybook 或单一 `/ui-preview` 对照页** 迭代到肉眼一致。**商标/图标资源**若为第三方专有，宜用自绘等价样式或占位图，避免直接搬运受版权保护的素材。

## 四、路由与 Layout 拆分（核心架构）

使用 **Route Groups**，避免工作台套营销 Navbar：

```mermaid
flowchart LR
  subgraph marketing [群组 marketing]
    MRoot[layout: 现有 Navbar Footer]
    MPages[ "/", "/pricing", "/docs", ... ]
  end
  subgraph workspace [群组 workspace]
    WRoot[layout: AppShell only]
    WHome["/home 或 /app 我的首页"]
    WWork["/work/[documentId] 文档编辑"]
  end
```

- **`(marketing)`**：迁移现有首页、pricing、blog 等，`layout` 保留 [Navbar](e:\my-project\slack-off-ai\slack-frontend/components/navbar.tsx)。
- **`(workspace)`**（命名可改为 `app` / `desk`）：**无**顶部营销 Navbar；全屏高度 `h-screen`、`overflow-hidden`；内含 **左侧栏 +（可选右侧 AI）** 的统一 `AppShell`。
- 参考链接形态：页面 **`/work/[documentId]`**（`documentId` 先用静态常量或 `[uuid]` 占位）。

默认主题：参考图多为**浅色工作台**；可将 workspace 默认设为 `light`，与根 [providers](e:\my-project\slack-off-ai\slack-frontend/app/providers.tsx) 里 `defaultTheme: 'dark'` 区分——workspace layout 内用 Providers 包裹或限定 `className`。

## 五、页面与组件清单（按截图分解）

以下为需实现的 **UI 模块**（功能延后 = 假数据 + `console.log` / `toast` 占位）。

### 5.1 工作台壳层 `AppShell`

- 左侧固定宽度（约 240–260px，可拖拽改宽为后续功能，UI 阶段可用固定 `w-[240px]` + 右侧 **仿拖拽条** 视觉）。
- 顶栏：划水品牌、字数「共 xxx 字」、右侧「AI 写作」、收藏/分享/发布/更多、语言、主题（与截图一致）。
- 底栏：左侧创建/更新时间；右侧 AI 免责 + token 文案（静态数字）。

### 5.2 侧栏 `WorkspaceSidebar`

- 用户/设置区、主导航（我的首页、搜索、收藏夹、我的分享、我的发布）。
- 「我的文档」可折叠 + **排序**按钮与下拉（默认排序、标题、创建/更新时间）——仅展开/选中态。
- 文档列表项：emoji/图标 + 标题 + 当前选中高亮；「+ 新建文档」。
- 底部：回收站、退出（退出可接 `window.confirm` 模拟参考图）。

### 5.3 我的首页（中央区）

- 大标题「我的首页」、搜索框 +「+ 新建文档」黑按钮。
- 「最近文档」「收藏文档」「分享文档」三段 **卡片栅格**；「更多...」链接样式。
- 空状态：「未找到内容」等。

### 5.4 搜索 / 收藏 / 分享 / 回收站 — **Modal 或全屏层**

- **搜索**：遮罩 + 居中白卡片、标题「搜索」、`Input`、「无结果」插图占位。
- **收藏**：表格骨架（标题 + 更新时间）；一行假数据可对齐截图。
- **我的分享**：空状态文案对齐参考。
- **回收站**：说明文案「保留 30 天」、搜索框、暂无数据。

以上可用 HeroUI v3 的 **Modal / Dialog、Listbox、Menu**（以实际 v3 API 为准）组合实现。

### 5.5 文档页 `/work/[documentId]`（核心）

- **中间编辑区**：标题行「请输入标题...」+ 图标装饰；正文为 **静态 HTML 或占位 div** —— **Tiptap 延到 UI freeze 之后**，避免编辑器行为干扰布局验收。
- **右侧 AI 面板**：可折叠；标题「AI 写作/聊天」、机器人插画占位、大号 `TextArea`、快捷链（续写/头脑风暴/写大纲/写总结）。
- **编辑区右侧竖条**：TOC 线、消息、绿色 AI、帮助等圆形按钮（堆叠）。
- 顶栏在「首页」与「文档」视图间可通过同壳不同 `slot` 切换。

建议组件目录示例（路径可按仓库习惯微调）：

- `components/workspace/app-shell.tsx`
- `components/workspace/workspace-sidebar.tsx`
- `components/workspace/workspace-header.tsx`
- `components/workspace/workspace-footer.tsx`
- `components/workspace/modals/search-modal.tsx`（等同理分离）
- `components/workspace/pages/home-overview.tsx`
- `components/workspace/pages/document-editor-mock.tsx`

## 六、分阶段交付（仅 UI）

1. **设计 Token + AppShell + Sidebar + Footer**：先跑通工作台全屏与不挤营销 Navbar。
2. **首页内容区**：标题、搜索、新建、三段卡片网格。
3. **Modal 家族**：搜索、收藏、分享、回收站 — 统一遮罩与圆角阴影。
4. **文档页三栏**：中间 mock 正文 + 右 AI 面板 + 浮动按钮列。
5. **主题与语言下拉**：仅切换 `next-themes` 与本地 state（文案切换可假）。
6. **对照走查**：用截图清单逐条打勾，修边距与颜色。

## 七、与远期《摸鱼AI完整开发方案》的衔接

- UI 阶段**不**引入 Prisma、Auth、Tiptap、Yjs、AI API。
- 路由 **`/work/[id]`** 与远期文档详情页一致时，后续只需把 `document-editor-mock` 换成真实 `TiptapEditor`，Modal 内接 API。
- 原方案 **Phase 1 layout** 可解读为「含工作台壳」——当前应拆成 **「UI shell 先行」** 与 **「API + 编辑器」** 两个子阶段，避免在一次迭代里揉杂。

## 八、风险与缓解

- **HeroUI v3 与模板代码混用**：老文档或 Copilot 可能生成 v2 API；缓解：每个新组件对照官方 v3，统一从 `@heroui/react` 导入。
- **根 layout 大改牵动现站**：仅用 Route Groups 可让 marketing 路由行为不变。
- **像素完全一致**：预留 0.5–1 轮专用于「间距/阴影微调」的子迭代。
