# 划水 AI 对齐：摸鱼 AI 全站中英文切换（修订方案）

## 0. 结论与范围（与用户要求对齐）

- **产品对齐**：划水 AI 英文入口为 [`/en`](https://www.huashuiai.com/en)，语言由 **URL 第一段** 表达；摸鱼 AI **100% 采用同一类产品行为**（不做「仅 Cookie、URL 不变」的折中）。
- **实现拍板**：采用 **路径前缀 `zh` | `en`**（与划水「显式语言段」一致；中文用 `/zh` 而非混用根路径，便于中间件、`lang` 属性与链接规则统一；根路径 `/` **重定向** 到 `/zh` 作为默认，与「中文站为主」一致）。
- **与竞品 URL 的备忘**：若划水中文站实际为 **根路径无 `/zh`**，则摸鱼采用 **`/zh/...`** 时在字符级 URL 上与竞品不完全一致；此为有意权衡（统一规则、减少例外）。英文仍为 **`/en/...`**，与竞品语义对齐。
- **技术选型**：**next-intl**（App Router + `middleware`），与官方维护的 `useTranslations` / `Link` / `usePathname` / `useRouter` 一体化，减少自研遗漏。
- **范围**：**营销区 + 工作台 + 全局错误与品牌相关 UI** 凡有用户可见文案或导航处，均纳入（见第 4 节盘点表）；**不包含**后端业务、未实现的 Auth 真实登出逻辑。
- **执行门禁**：本文档更新供评审；**待用户明确确认后再进入编码实施**。

---

## 1. 划水 AI 行为归纳（需求对照）

| 划水 AI 表现                    | 摸鱼 AI 对应实现                             |
| ------------------------------- | -------------------------------------------- |
| 英文站 URL 含 `/en`             | 所有页面形如 `/en/...`                       |
| 语言可分享、可收藏              | 同款；切换语言 = 换 locale 段并保持后续 path |
| （推断）默认/另一语言有独立 URL | `/zh/...` + `/en/...`，`/` → 重定向 `/zh`   |

说明：未抓包划水是否用 `next-intl`，**产品层面对齐 URL 模式即可**；实现层以 next-intl 为准。

## 2. 架构（实现后数据流）

```mermaid
flowchart TB
  subgraph edge [Edge Middleware]
    detect[Accept-Language optional]
    cookie[Cookie NEXT_LOCALE]
    redirect[Slash redirect to defaultLocale]
    prefix[Ensure first segment is locale]
  end
  subgraph app [App Router]
    locLayout["app/[locale]/layout.tsx"]
    locLayout --> marketing["(marketing)/..."]
    locLayout --> workspace["(workspace)/..."]
    messages[Load messages/zh|en]
    locLayout --> messages
  end
  Request --> edge
  edge --> app
```

要点：

- **`app/[locale]/layout.tsx`**：设置 `<html lang={locale === 'zh' ? 'zh-CN' : 'en'}>`（从当前根 `lang='zh-CN'` 改为动态），并接入 `NextIntlClientProvider`（仅客户端子树需 message 时）。
- **根 `app/layout.tsx`**：仅保留与 next-intl 文档兼容的最薄外壳（或合并进 `[locale]`，以官方示例为准，避免双 `<html>`）。
- **主题**：继续使用现有 `@teispace/next-themes` [`slack-frontend/app/providers.tsx`](slack-frontend/app/providers.tsx)，放在不变的全局或 `[locale]` layout 中均可，**不与语言强耦合**。

## 3. 实现步骤（有序、可执行）

1. **依赖与配置**
   - `pnpm add next-intl`
   - 新增 `i18n/request.ts`（或项目约定路径）、`routing.ts`：`locales = ['zh','en']`，`defaultLocale = 'zh'`。
   - **[`slack-frontend/next.config.mjs`](slack-frontend/next.config.mjs)**（仓库当前为 `.mjs`，不是 `next.config.ts`）按 next-intl 文档接 `createNextIntlPlugin`（若官方要求）。

2. **Middleware**
   - 新建 [`slack-frontend/middleware.ts`](slack-frontend/middleware.ts)：
     - Locale 检测与前缀补全；`/` → `/zh`；
     - **Matcher 排除** `_next`、静态资源、`favicon`、`/api`（未来若有）。
   - 与 `Navbar` 中「首页极简顶栏」的路径判断同步改为含 locale（见第 5 节）。

3. **目录迁移**
   - 将 `slack-frontend/app/(marketing)`、`slack-frontend/app/(workspace)` 迁至 **`app/[locale]/(marketing)`**、**`app/[locale]/(workspace)`**（保留 route group 名称，减少 diff）。
   - 动态路由 `app/(workspace)/work/[documentId]/page.tsx` 一并变为 `/[locale]/work/[documentId]`。

4. **文案与键**
   - `messages/zh.json`、`messages/en.json`：扁平或嵌套命名空间（如 `Marketing.home.title`、`Workspace.sidebar.newDoc`）。
   - 页面级优先 `getTranslations`（RSC）；客户端组件 `useTranslations`。
   - **Mock 数据**：见第 4 节 **`lib/workspace-mock.ts`** 与 todo **`mock-strategy`**，不能只改 UI 字符串而留整套中文 mock。

5. **导航与站内链接**
   - 所有 **`next/link`、`useRouter`、`redirect`** 改为 next-intl 的 **`Link`、`useRouter`**（不推荐手写 `` `/${locale}` `` 字符串）。
   - [`slack-frontend/config/site.ts`](slack-frontend/config/site.ts)：`navItems` **仅存无前缀 pathname**（如 `/docs`），由 next-intl `Link` 自动加 locale。
   - 按 **第 5.2 节站内裸链枚举** 逐项替换，实施后 **`pnpm build` 前用 grep 复核**（见第 6 节）。

6. **pathname 与路由感知（极易漏）**
   - 引入 `[locale]` 后，浏览器 pathname 为 `/zh/work/xxx`，凡 **`pathname.startsWith('/work/')`**、**`slice('/work/'.length)`** 的逻辑会失效。
   - **必须**改为 next-intl 提供的 **`usePathname()`（不含 locale 前缀）**，或统一封装「去掉首段 locale」的工具函数。涉及文件见 **第 5.1 节**。

7. **语言切换组件**
   - 单一 **`LanguageSwitch`**：`useLocale()` + `usePathname()`（next-intl）→ `router.replace` 切换 locale、**路径后缀不变**。
   - 移除 [`slack-frontend/components/workspace/workspace-header.tsx`](slack-frontend/components/workspace/workspace-header.tsx) 内重复 `locale` state；主题下拉「明亮/黑暗/系统」等进词典。

8. **营销页特例**
   - [`slack-frontend/components/marketing-footer.tsx`](slack-frontend/components/marketing-footer.tsx)、[`slack-frontend/components/navbar.tsx`](slack-frontend/components/navbar.tsx)：改用 **next-intl 的 `usePathname()`**（通常为 **去掉 locale 前缀** 的路径）。本地化营销首页在该 hook 下一般为 **`pathname === '/'`**，与当前「隐藏 Footer / 极简 Navbar」语义对齐；**勿**继续用 `next/navigation` 的 `usePathname()` 去拿含 `/zh`、`/en` 的原始路径（除非有意为之并全仓统一）。

9. **SEO / 无障碍**
   - `metadata`：`generateMetadata` 随 locale 切换 `title`/`description`（至少首页与全局 template）。
   - 可选：`alternates.languages` / hreflang，与 **`/zh` 作为中文 canonical** 策略一致，避免与「根路径仅重定向」混淆。

10. **后续路由模板**
    - 当前无 [`global-error.tsx`](slack-frontend/app/global-error.tsx)、[`not-found.tsx`](slack-frontend/app/not-found.tsx)；若新增，须默认接入 `[locale]` 或共用词典。

## 5. 与现有逻辑的冲突点（必改）

### 5.1 `[locale]` 前缀下的 pathname 解析（高优先级）

真实 URL 为 **`/zh/work/xxx`**、`/en/work/xxx` 时，下列代码中 **`pathname.startsWith('/work/')`**、**`pathname.slice('/work/'.length)`** 会 **全部失效**，导致文档页不识别、侧栏不高亮、`documentId` 错误：

- [`slack-frontend/components/workspace/app-shell.tsx`](slack-frontend/components/workspace/app-shell.tsx)
- [`slack-frontend/components/workspace/workspace-sidebar.tsx`](slack-frontend/components/workspace/workspace-sidebar.tsx)
- [`slack-frontend/components/workspace/workspace-footer.tsx`](slack-frontend/components/workspace/workspace-footer.tsx)

**要求**：统一使用 **next-intl 的 `usePathname()`（不含 locale 前缀）**，或单一工具函数从原始 pathname 去掉首段 locale 后再判断 `/work/`。

### 5.2 站内裸链枚举（替换 next-intl Link 时逐项划掉）

| 文件 | 当前典型写法 |
| -------------------------------------- | ------------- |
| `slack-frontend/app/(marketing)/page.tsx` | `/home` |
| `slack-frontend/components/navbar.tsx` | `/`（Logo）、`siteConfig.navItems` 的 href |
| `slack-frontend/components/workspace/workspace-sidebar.tsx` | `/home`、`` `/work/${doc.id}` `` |
| `slack-frontend/components/workspace/workspace-document-card.tsx` | `` `/work/${doc.id}` `` |
| `slack-frontend/components/workspace/brand-mark.tsx` | 默认 `/home` |
| `slack-frontend/components/workspace/workspace-footer.tsx` | `/pricing` |

实施后验收：对仓库执行 grep，确保无遗漏的 **`href='/...'`** / **模板字符串手写 `/work/`、`/home`**（middleware、注释、允许列表除外）。

### 5.3 其他

- **`pathname === '/'`（原生）**：`MarketingFooter`、`Navbar` 首页分支 —— 随本地化首页语义更新（与 next-intl 的 pathname 行为对齐，见步骤 8）。
- **intl-messageformat**：已在 [`slack-frontend/package.json`](slack-frontend/package.json) 但未使用 — **可选用**；非必须，`next-intl` 默认 ICU 已够用。
